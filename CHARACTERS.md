# CHARACTERS.md — The SPOF Character Style Bible

The sprite counterpart of PROMPTS.md's background prompt prefix. This is
M3.5's lasting artifact: every character from M4 on is minted inside that
act's own content thread **using this document**, calibrated against the
existing cast. There is no recurring character milestone. (A character
that breaks this process — unique animation, set-piece scale — earns a
mini dedicated pass; exception-driven, never calendar-driven.)

**Reference plate:** `assets/source/character_reference.png` — the cast
at 1x and 3x (Mel ×4 outfits, Gary, Darlene, Merle) plus Mel's
right-facing walk row. Regenerated by every sprite build; if the plate
and this file disagree, the plate wins.

## How characters are made (the decision)

Characters are **hand-pixeled, not generated**. Backgrounds come from one
image generator because painterly rooms survive quantization; sprites do
not — at 32x64 with 6-frame cycles the whole problem is frame-to-frame
identity, which generators can't hold and hand-cleaning would re-draw
anyway. So the pixels are typed by hand:

- Art source = **grid files**: multiline strings, one character per
  pixel, in one Python module per cast member under `tools/sprites/chars/`.
- `tools/sprites/build_sprites.py` compiles grids into the PNG sheets in
  `assets/sprites/` (plus the reference plate). The compiler only
  assembles what the artist typed — parse, layer, mirror, pack. It never
  invents pixels. Deterministic: same grids, same bytes.
- `data/sprites.json` registers each sheet for the engine (frame size,
  anchor, fps). Rooms place NPCs; game.json maps Mel's outfits. All data
  — see `data/schema.md` "Sprites".

Rebuild after any grid edit:

    python3 tools/sprites/build_sprites.py
    # iterate with previews on room-floor swatches:
    python3 tools/sprites/build_sprites.py --only <name> --preview /tmp/prev

## The one style authority

`tools/sprites/chars/_master_palette.py` is the single palette table for
the entire cast — the sprite counterpart of the one-background-generator
rule. House law:

- **Flat 3-tone ramps** (dark / mid / light) + the shared warm outline
  (`outline`, near-black). **No dithering on sprites** — flat Sierra
  figures over painterly rooms is the authentic VGA look, and it survives
  the 320x180 phone squint.
- Warm-leaning, slightly muted values that sit inside the treated
  128-color rooms. Every character owns **one signature color pop**:
  Mel slate (+ teal headphones), Gary powder-blue polo, Darlene teal +
  white apron, Merle rust flannel + gold cap patch. Pick a new
  character's pop against the whole cast lineup, not in isolation.
- New characters reuse existing ramps first. A genuinely new ramp is
  added to the master palette file, never inlined — the smoke suite
  samples exact ramp RGB values off the canvas as render evidence, and
  palette drift would quietly break that.
- Ground shadow is **baked into the art** (the `shadow` alpha color) so
  it never slides during walk frames. Characters with no visible feet
  (Darlene) get no shadow.

## World scale & proportions

- Figures are **~48-50 px tall standing** (heads ~12 px: chunky
  VGA-Sierra proportions, roughly 1:4). This was fitted against door
  heights and the reserved NPC polygons; do not eyeball a new size.
  There is no depth scaling — walkable bands are shallow, one size per
  room reads fine.
- Mel's frame is 32x64 (figure bottom-anchored, headroom spare). NPC
  frames are whatever the design needs (Gary 40x52 fits the hose arc;
  Darlene 30x38 waist-up; Merle 30x54 seated). True scale, always —
  sheets are drawn at final pixel size, never downscaled.
- **Anchor** = the frame pixel that lands on the world (x, y): feet
  between the heels for standers, boot soles for sitters, the waist-cut
  row for counter NPCs. Declared per sheet in `data/sprites.json`.

## Sheet conventions (the engine contract)

- Columns = frames; **col 0 is idle**, the rest are the cycle.
- **Walker** (4 rows): direction rows in fixed order down/left/right/up;
  cols 1+ are the 6-frame walk. Left row = mirrored right row (the
  compiler flips; nobody at Cumulonimbus notices the glasses switch
  temples).
- **Static NPC** (1 row): cols 1+ are the 2-3 talk frames. Talk frames
  are *gesture* frames, not mouth flaps — at 12 px heads, arms act:
  Gary's free arm rises to a bellow, Darlene's pot orbits + free hand
  wave, Merle's saucer lifts. The engine cycles them for a beat whenever
  TALK/ask lands on the hotspot; nothing to author per-line.

## The walk cycle (Mel spec, for future walkers)

Six phases per side, the classic contact–recoil–pass twice over:

1. contact (wide stride, front heel down, back toe down)
2. recoil (gather, weight over planted leg, **upper body drops 1 px**)
3. pass (planted leg vertical, swing foot lifted under the body)
4-6. the same with legs swapped

Arms swing opposite the near leg (poses: fwd / down / back). Far limbs
are drawn in ramp-shadow tones for depth (the compiler recolors the same
arm grids; far leg uses dark logical chars). Front/back walks are a
6-frame scissor from 3 hand-drawn leg grids + their mirrors, plus the
same 1 px upper-body bob. ~9 fps.

## Outfits (Mel's part-swap system)

Mel is parts: heads x3 directions, hoodie uppers, coat uppers (longer —
the hem overlays the leg grids), and **leg grids drawn once in logical
characters** with per-outfit color keys (joggers key / khaki key). The
same silhouette compiles to gray joggers + white sneakers or khaki
slacks + brown shoes (+ a crease highlight only the khaki key reveals).
Four sheets from 2 uppers x 2 legsets:

    mel_base = hoodie + joggers        mel_pants = hoodie + khaki
    mel_coat = coat + joggers          mel_coat_pants = coat + khaki

Worn-state flags select the sheet through the game.json outfit map
(first-match, standard conditions — schema.md "Sprites"). A future
outfit (the backup hoodie's cold-aisle beat) = new part grids, compile a
new sheet combination, one registry row, one outfit-map entry. Zero
engine work.

## Make-a-character (the repeatable process)

1. **Read their LOOK text first.** The room JSON already describes them;
   the sprite illustrates the narrator, never contradicts him. If the
   room isn't written yet, write the LOOK line first — words are the
   cheaper iteration loop.
2. **Pick silhouette + signature pop against the lineup.** Open the
   reference plate. The new character must be tellable from every
   existing cast member at 1x by outline alone (hose/apron/saucer-scale
   props are silhouette). Choose ramps from the master palette; add a
   ramp only if the design truly needs one.
3. **Copy the nearest module** in `tools/sprites/chars/` (Gary =
   standing, Darlene = behind-furniture waist-up, Merle = seated) to
   `<name>.py`. Register it in `CHAR_MODULES` in `build_sprites.py`.
4. **Draw the idle grid** (body base + head), sized per World Scale
   above. Type pixels; build with `--only <name> --preview`; look at it
   on the room's floor swatch; iterate. Then draw the 2-3 talk-gesture
   frames as small overlay parts (an arm, a prop pose) — not full
   redraws.
5. **Behind furniture?** End the art at the occluding edge (Darlene's
   waist-cut) and anchor that row so it tucks behind the furniture's FAR
   edge — on the near edge she reads as standing in front with her legs
   missing. **Seated?** Sit them on painted furniture; anchor at the
   lowest body pixel. Keep the anchor y **at or above the top of the
   walkable band** (draw order sorts by anchor y, so Mel still passes in
   front), and put standers' feet ON a painted ground line — a foot row
   floating above the lawn/sidewalk boundary reads as levitation.
6. **Register + place:** add the sheet to `data/sprites.json` (frame
   size, anchor, talkFps); add `"sprite": { "use": "<id>", "at": [x, y] }`
   to their hotspot; fine-tune `at` against `?dev=1` or a devshot.
7. **Extend the smoke suite:** NPC id present in the room, one ramp
   color sampled inside their box, talk cycle runs on TALK. Three
   checks, same pattern as Gary/Darlene/Merle.
8. **Rebuild everything** (`build_sprites.py` with no flags) so the
   reference plate picks up the newcomer, and eyeball the lineup once —
   that's the calibration step, and it's the whole point.
